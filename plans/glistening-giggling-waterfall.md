# Level 01 に CLAUDE.md 作成ミッション追加 & 問題文リライト

## Context
現在のワークブックは Level 01 が Mission 1-A（Instagram投稿文生成）から始まるが、Coworkの基本操作（ファイルを読ませる → ファイルを作らせる）を最初に体験させるステップがない。
フォルダ内のファイル内容を説明する CLAUDE.md を最初に作らせることで、Coworkの基本動作を学びつつ、プロジェクト全体の理解も深まる。

## 変更内容

### 1. 新ミッション「Mission 1-0」を追加（Step 0 の直後、1-A の前）

**ミッション概要:**
- フォルダ内のファイル（product_spec.txt, customer_reviews.csv, sales_monthly.csv, competitor_info.txt）を全て読み、各ファイルの内容を説明する CLAUDE.md を作成させる
- ツール: Cowork
- 難易度: ★☆☆☆☆（最低）
- 成果物: CLAUDE.md

**シナリオ案:**
新商品「ビタグロウ セラム エクストリーム」のプロジェクトフォルダを引き継いだ。中にファイルが4つあるが、説明がない。Coworkにフォルダ内のファイルを読ませて、各ファイルの内容を説明する CLAUDE.md を作成してもらう。

**タスク:**
- フォルダ内の全ファイルをCoworkに読ませる
- 各ファイルの内容（ファイル名・形式・概要・含まれる項目）を説明する CLAUDE.md を作成させる
- 後から見た人がすぐにプロジェクトの全体像を理解できるようにする

**着目ポイント:** 「ファイルを全部読んで」と雑に頼むのと、「各ファイルの形式・概要・項目を整理して」と条件を指定するのでは結果が違う

**模範プロンプト（Cowork）:**
```
フォルダ内のすべてのファイルを読み込んで、
プロジェクトの CLAUDE.md を作成してください。

【CLAUDE.mdに含める内容】
- プロジェクト概要（商品名・目的）
- 各ファイルの説明（ファイル名、形式、概要、主要な項目）
- ファイル間の関係性

【出力形式】
Markdown形式で、見出し・箇条書きを使って整理
```

**模範プロンプト（チャットAI）:**
```
以下の4ファイルの内容をもとに、プロジェクトのCLAUDE.mdを作成してください。

--- product_spec.txt ---
（内容を貼り付け）
--- customer_reviews.csv ---
（内容を貼り付け）
...
```

**何が違う？:** Coworkはフォルダ内のファイルを自分で全部読みに行ける。チャットは4ファイル分の内容を全部コピペする必要がある。ファイル数が多くなるほどCoworkの優位性が際立つ。

**マネージャーの視点:** プロジェクトを引き継いだとき、最初にCLAUDE.mdを作らせるのは実務でも有効。新メンバーのオンボーディング時間を大幅に短縮できる。
